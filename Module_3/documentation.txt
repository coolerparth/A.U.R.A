
# Project Documentation: Resume Scoring Engine

**Last Updated**: March 19, 2026

## 1. Project Overview

The Resume Scoring Engine is a comprehensive Applicant Tracking System (ATS) designed to 
automate the evaluation of candidate profiles against job descriptions. It is composed of two primary systems working in tandem:

1.  **Data Aggregation Pipeline**: A powerful data extraction system that builds a holistic digital profile of a 
candidate by scraping information from multiple online sources (GitHub, LeetCode, etc.).
2.  **ATS Scoring Engine**: A sophisticated analysis engine that scores the aggregated candidate profile against a given 
job description using a multi-faceted, AI-powered approach.

This document provides a complete technical breakdown of the project's architecture, logic, and code.

---

## 2. System Architecture & Workflow

The project follows a two-stage workflow:

**Stage 1: Data Aggregation**
- **Input**: A basic JSON file (`individual1.json`) containing the candidate's name and links to their online profiles.
- **Process**: The `pipeline.py` script orchestrates a series of scrapers located in the `extraction/` directory. 
These scrapers fetch detailed information from platforms like GitHub, LeetCode, and Codeforces.
- **Output**: A single, consolidated `resume.json` file that contains all the aggregated data, forming a comprehensive digital resume.

**Stage 2: Scoring & Analysis**
- **Input**: The `resume.json` file (from Stage 1) and a `job_description.json` file.
- **Process**: The `ats_scoring_engine.py` script analyzes the two JSON files, calculating a compatibility score based on various metrics.
- **Output**: A detailed `ats_score.json` file containing the final score and a full breakdown of how it was calculated.

## 3. Detailed Logic & Code Breakdown

### 3.1. Data Aggregation Pipeline

The core of the data aggregation system is the `pipeline.py` script, which manages the entire extraction and merging process.

#### `pipeline.py` - The Orchestrator

This script executes the data aggregation in three sequential steps:

1.  **Extract Links**: It reads the initial `individual1.json` and extracts the usernames and profile URLs for various platforms.
2.  **Extract Data**: It calls the `extract_all_profiles` function from `extraction/extract_all.py` to run the individual scrapers.
3.  **Merge Data**: It calls the `merge_all_data` function from `extraction/merge_data.py` to combine the initial data 
with all the scraped data into the final `resume.json`.

**Code Snippet (`pipeline.py`):**

### 3.2. ATS Scoring Engine

The `ats_scoring_engine.py` script is the heart of the project's analytical capabilities. It takes the `resume.json` 
and `job_description.json` and produces a detailed score.

#### Core Scoring Logic

The engine calculates a final score by breaking down the analysis into four main components and applying them to 
different sections of the job description.

**1. Four Core Scoring Metrics:**

*   **Keyword Score (`keyword_score`)**: Measures the direct overlap of normalized keywords between the resume and a 
job description section.
    *   **Logic**: `(Number of Matching Keywords) / (Total Unique Keywords in Job Section)`
*   **Semantic Score (`semantic_score`)**: Uses the `all-MiniLM-L6-v2` sentence-transformer model to compare the 
contextual meaning of the texts.
    *   **Logic**: Calculates the cosine similarity between the vector embeddings of the resume and job section texts.
*   **Numeric Score (`numeric_metrics_score`)**: Extracts and compares numerical requirements (e.g., "5+ years of experience").
    *   **Logic**: `(Number of Numeric Requirements Met) / (Total Numeric Requirements)`
*   **Job Title Score (`job_title_score`)**: Compares the candidate's current job title to the one in the job description.
    *   **Logic**: Returns 1.0 for an exact match, or a partial score based on word overlap.

**2. Profile-Based Scoring:**

The engine uses two different "profiles" to weight the four core metrics differently depending on the importance 
of the job description section.

*   **`KEYWORD_BIASED_PROFILE`**: Used for critical sections like "Required Qualifications." It gives more weight to the `keyword_score`.
    *   `{"exact_keyword": 0.75, "semantic": 0.10, ...}`
*   **`SEMANTIC_BIASED_PROFILE`**: Used for broader sections like "Responsibilities." It prioritizes the `semantic_score`.
    *   `{"semantic": 0.75, "exact_keyword": 0.10, ...}`

**3. Dynamic Weight Redistribution (`redistribute_weights_for_zeros`):**

This is a crucial feature for ensuring fairness. If any of the four core metrics results in a score of zero, 
the system does not let it unfairly drag down the total.

*   **Logic**: When a component scores zero, its weight is not counted as a complete loss. Instead, 80% of 
its assigned weight is redistributed proportionally among the other, non-zero components. The remaining 20% is treated as a penalty. 
This prevents a single missing element from nullifying an entire section's score.

**4. Final Score Calculation:**

The final ATS score is a weighted average of the scores from each major job description section 
(`required_qualifications`, `responsibilities`, etc.).

*   **Logic**: `Final Score = (Score_Section1 * Weight_Section1) + (Score_Section2 * Weight_Section2) + ...`

## 4. How to Run the Project

### Prerequisites

- Python 3.x
- `pip` for installing packages

### Setup

1.  **Clone the repository:**
    ```bash
    git clone <repository_url>
    cd Resume_Scoring_Engine
    ```
2.  **Create a virtual environment:**
    ```bash
    python -m venv .venv
    source .venv/bin/activate  # On Windows: .\.venv\Scripts\activate
    ```
3.  **Install dependencies:**
    ```bash
    pip install -r requirements.txt
    ```
4.  **Download NLP model data (first time only):**
    The `sentence-transformers` model will be downloaded automatically on its first run.

### Execution

1.  **Run the Data Aggregation Pipeline:**
    (Ensure you have an `individual1.json` file ready)
    ```bash
    python pipeline.py
    ```
    This will generate the `resume.json` file.

2.  **Run the ATS Scoring Engine:**
    (Ensure you have a `job_description.json` file ready)
    ```bash
    python ats_scoring_engine.py --job job_description.json --resume resume.json --output ats_score.json
    ```
    This will generate the final `ats_score.json` with the detailed results.
